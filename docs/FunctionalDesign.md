## 0. 基本設計書 §8 未確定事項の確定

### 0-1. 座標と固定小数（決定論の基盤）
- 位置・速度・距離はすべて**ミリ単位の整数**で保持する（`POSITION_SCALE = 1000`。表示座標 = 内部値 / 1000）。アリーナは内部 `0..1000000`（=0..1000 表示）。
- **整数平方根 `isqrt(n)`**：2分探索で floor(√n) を返す（決定論的）。距離が必要な箇所のみ使用。
- 乱数・浮動小数・時刻・map 反復順は一切使わない。

### 0-2. 2D 移動の整数化
目標 (tx,ty)、現在 (sx,sy)、1ティックの歩幅 `step`（ミリ）に対し：
```
dx = tx - sx; dy = ty - sy
dist = isqrt(dx*dx + dy*dy)
if dist == 0: 移動なし
elif dist <= step: 目標へ到達（sx,sy = tx,ty）
else: sx += dx*step/dist; sy += dy*step/dist   // 整数除算（切り捨て・決定論的）
```
retreat は (tx,ty) を「敵の反対方向の点」に置き換えて同式を使う。

### 0-3. 速度・重量（固定小数を整数で）
- `excessWeight = max(0, totalWeight - chassis.weight)`
- `step(ミリ/tick) = max(SPEED_MIN*1000, chassis.baseSpeed*1000 - 200*excessWeight)`（200 = 0.2×1000）
- オーバーヒート中は `step = step * OVERHEAT_SPEED_MULT`（=step/2）。

### 0-4. 初期配置（1v1）
- 挑戦者＝(150000, 500000)（表示 150,500）、相手＝(850000, 500000)（表示 850,500）。互いに向き合う。

### 0-5. keepDistance（カイティング）保持帯
- 基準距離 = 自武器の射程 `R`。`dist < R*8/10` → retreat、`dist > R` → approach、その間 → stop。
- 武器が無い場合は approach にフォールバック。

### 0-6. 条件 op の意味
- `enemyDistance`：`inRange`=dist ≤ 自武器射程、`outRange`=その否定、`lt/gt`=value（表示単位）と比較。
- `selfHp`：`hp*100/maxHp` と value(%) を lt/gt 比較。
- `selfShield`：`exists`=shield>0／`none`=shield==0。
- `selfHeat`：`lt/gt`=value と比較、`nearOverheat`=heat ≥ OVERHEAT_THRESHOLD*8/10。
- `selfBattery`：その tick の利用可能電力に対する消費見込みの割合で lt/gt（段階1ではほぼ常に余裕＝high 側。意味を持つのは案B＝段階2）。
- `dashReady`：dashCd==0（段階2）。

### 0-7. パーツの「稼働」と電力消費
- **行動時消費**：パーツはその行動を実行したティックのみ電力を消費（段階1の素体・武器は待機時消費 0）。
  - 武器発射時：`実消費 = powerCost + powerCost*heat/200`（heat 比例。0.5×heat/100 = heat/200）。
- `利用可能電力(tick) = batteryCapacity - excessWeight/10`。
- 段階1：案A でビルド時に「武器を撃てる」基準（`powerCost ≤ 利用可能電力`）を検証。**ランタイムの熱による超過は段階1では許容（行動は止めない）**。電力不足による抑制は段階2（案B）で実装。

### 0-8. 重なり処理
- 移動適用後、両者の距離が `MIN_SEP=40000`（表示40）未満になる場合、接近した側を接近ベクトル上で `MIN_SEP` の位置まで戻す（同座標化・追い越しを防ぐ）。

### 0-9. 段階1 sim 定数（確定）
`POSITION_SCALE=1000, SPEED_MIN=3, OVERHEAT_SPEED_MULT=1/2, MIN_SEP=40000`、その他は BasicDesign §3.5（HEAT_DECAY=2 ほか）に従う。ティック30/秒・最大1800。

## 2. F1 ビルド作成
**段階1の編集範囲**：素体は「Balanced」固定、武器スロットに「Starter Cannon」を装着（1個）、行動ルールをチャネル別に編集。

- **入力**：owner名、ロボ名、（武器の装着/取り外し）、ルール編集。
- **検証（登録時・サーバー）**：ステータス正値、`Σ slotCost ≤ slots`、案A の電力（0-7）、Ruleset の条件/行動が既知種別。
- **出力**：登録成功で Robot（id付き）を返し名簿入り。
- **制約表示**：使用スロット / 総重量 / 実効速度 / 電力収支をリアルタイム表示。

### ルールエディタ（段階1）
- チャネル（移動／武器）ごとにルールを**リスト表示**（上が高優先）。「ルール追加」で〔条件（複数AND）〕＋〔行動〕を選択。並べ替えで優先順位変更、削除可。
- 行動の選択肢は装備に依存（武器なしなら武器チャネルは hold のみ）。
- デフォルト動作（移動=approach／武器=hold）は常に末尾に暗黙適用。

## 3. F2 名簿・相手選択
- `GET /api/robots` で一覧表示（名前・owner・主要ステータス）。各行に「挑戦」。
- 自分の登録済みロボ（挑戦者）を選択した状態で相手を選ぶ。

## 4. F3 挑戦・観戦
- 「挑戦」→ `POST /api/challenge {challenger_id, opponent_id}` → Battle（Replay含む）取得 → **Canvasで再生**。

### 観戦描画仕様（Canvas）
- 表示座標 = 内部ミリ / 1000 をキャンバス幅にスケール。
- 各ロボ：円（青=挑戦者/赤=相手）。上に **HPバー**、その上に **シールドバー**（あれば）、**熱ゲージ**（色が熱で変化、オーバーヒートで点滅）。
- 攻撃イベント：source→target に攻撃色の線を1ティック描画。破壊イベント：対象を×表示。
- 再生：30tick/秒。終了時に「勝者: 名前（reason）」表示。再生コントロール（再再生）。

## 5. F4 受信箱
- `GET /api/inbox?owner=` で、自分のロボが挑まれた対戦を一覧（挑戦者名・防衛側視点の勝敗）。各行「観戦」で F3 の描画を再利用して再生。

# 段階2 機能設計（戦闘モデルの進化）

参照：BasicDesign §9「2-G」。hitscan を発射体に置換し、遮蔽物・マップ拡大を追加する。座標は引き続きミリ整数。

## S2-0 データ追加
- `WeaponSpec.projectileSpeed`（units/tick）。
- `Frame.projectiles: [{x,y,source}]`（ミリ座標・描画用）。
- `Replay.obstacles: [{x,y,w,h}]`（ミリ）、`Replay.arenaW/arenaH`（ミリ）。
- sim 内部の発射体：`{x,y,vx,vy,source,damage,traveled,range}`（traveled で飛距離管理）。

## S2-1 マップ・初期配置・遮蔽物（ミリ・点対称で公平）
- アリーナ 1,600,000 × 1,600,000。
- 初期配置：**斜め**（中心(800000,800000)について点対称）。挑戦者 (200000,650000)／相手 (1400000,950000)。斜めにすることで approach に X・Y 両成分が出て、壁ずりで回り込める。
- 遮蔽物5個（点対称）：
  - A前カバー (315000,640000,70000,110000)／B前カバー (1215000,850000,70000,110000)
  - 中央 (770000,760000,60000,80000)／(550000,500000,60000,140000)／(990000,960000,60000,140000)
  - 各スポーン前カバーが初期配置の射線を遮る（数値は小方針・調整可）。

## S2-2 1ティックの処理順（改訂）
1. 減衰（熱・クールダウン・オーバーヒート回復）。
2. スナップショットからチャネル評価（移動・武器の行動決定）。
3. ロボ移動を算出 → **遮蔽物クリップ（S2-3）** → 適用＋重なり防止。
4. 発射：武器 fire ∧ `weaponCd==0` ∧ 非オーバーヒート で、自機位置に発射体生成。速度＝スナップショットの敵位置方向 × projectileSpeed（`stepToward` と同じ整数方向算出）。damage＝発射時 power、range＝weapon.range、weaponCd/熱を設定。
5. 全発射体を移動（x+=vx, y+=vy, traveled+=projectileSpeed）。各発射体を判定：traveled≥range で消滅／いずれかの遮蔽物内（点が矩形内）で消滅／**相手**ロボ中心と HitRadius 内で命中（シールド→HP）＋消滅。
6. フレーム記録（ロボ状態＋発射体）。
7. 決着判定（§3.6 と同じ）。
- 同時解決は維持（行動決定はスナップショット基準）。命中はもはや即時でなく「弾の到達」で起きる。

## S2-3 遮蔽物による移動制限（壁ずり）
点が矩形内か：`ox ≤ x ≤ ox+w` かつ `oy ≤ y ≤ oy+h`（ロボ中心で判定）。移動は**軸別に適用（壁ずり）**：
1. 目標座標 (nx,ny) がどの遮蔽物内でもなければそのまま適用。
2. 内なら、**X だけ動かした (nx, oldY)** が遮蔽物外なら採用。
3. それも内なら、**Y だけ動かした (oldX, ny)** が遮蔽物外なら採用。
4. どちらも内なら停止（その tick は動かない）。
これで矩形の縁に沿ってスライドし、凸障害物（本作の矩形）を回り込める。すり抜けはしない。

## S2-4 攻撃判定の変更
- §3.4 の hitscan を廃止。発射体ライフサイクル（S2-2）に置換。条件 `enemyDistance inRange` は引き続き weapon.range 基準で「いつ撃つか」をアルゴリズムが判断するのに使う。

## S2-5 描画（観戦・F3 更新）
- スケールは 1,600,000 基準に変更。
- 遮蔽物：塗りつぶし矩形（灰）。発射体：小さな点（source 色）。ロボ・各バーは段階1と同様。

## S2-6 段階2 定数（小方針・調整可）
`HitRadius=18000`（18units）、Starter Cannon `projectileSpeed=40`（=40000ミリ/tick）。

## S2-7 段階2で実装しないもの
リード射撃（未来位置予測）／カバー退避などの条件・行動パレット拡張／移動のスライド処理。装甲・ダッシュ・素体・防御・武器バリエーション・UIタブ化は段階3。
