# 開発ルール

本リポジトリの開発で守るルール。アジャイルで反復しながら、技術的障壁とその解決策を記録として積み上げることを目的とする。

## R1. リポジトリ隔離
- 本プロジェクトのファイルは、すべてこのリポジトリ内にまとめる。
- 他のプロジェクトのファイルと混在させない。

## R2. 自己完結
- このリポジトリ単体で完結させる。
- 本プロジェクトに無関係なファイルはコミットしない。

## R3. 単体テスト
- 機能実装の単位ごとに単体テストを作成する。
- テストは実装と並走させ、こまめに積み上げる。

## R4. 進捗の明示
- README 等に、進捗状況と現在の課題・問題点を細かく記載し続ける。
- **直面した技術的障壁と、その解決策を必ずセットで記録する**（本プロジェクトの中心的な記録方針）。

## R5. 静的解析・コードスタイル
- 静的解析ツール（リンター／フォーマッター等）を導入する。
- コードスタイルを設定ファイルでルール化し、自動で整える。
- （具体的なツールは技術スタック確定時に選定する。例: TypeScript なら ESLint + Prettier）

## R6. 可読性
- 誰が読んでも分かりやすいコードを書く。
- 命名・関数分割・構造を重視し、読み手に優しい実装を心がける。
